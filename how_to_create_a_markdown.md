 # 准备工作
 vscode中有各式各样的markdown插件，但其实，如果我们只是有一些基础需求，即使不安装任何插件也是可以的，因为vscode默认支持markdown的核心功能。

 如果我们的需求是自动目录、语法检查等需要满足的高级功能，那么可以安装插件来提升效率和功能。

 那还说什么呢，快快动动手指创建.md文件！
 # 有关标题
如果想要在md文件中添加各级标题，只需要在标题文本的前面加入 **“# ”——井号＋空格**。
- 若干#对应若干级标题。（例如，一个#对应一级标题，以此类推）
- 最多支持6级标题，超出6个#变为正文。
- 如果#后方无空格则会变为正文。
- 除此之外，对于一级标题和二级标题，也可以采用下划线的形式。
    - 【一级标题文本】->【换行】->【若干等于号=】
    - 【二级标题文本】->【换行】->【若干减号-】

# 有关正文
## 段落间换行
需要空一行来进行分隔，单换行仅视为空格。

## 强调
如果我们需要在正文中强调词语或语句，你可以选择：

- 斜体：强调部分的前后分别加单个星号（*）或单个下划线（_）
- 加粗：强调部分的前后分别加两个星号（**）或两个下划线（__）
- 斜体加粗：强调部分的前后分别加三个星号（***）
- 删除：强调部分的前后分别加两个波浪号（~~）

## 代码

行内代码前后分别用一个反引号（`）包裹。

代码块有两种方式实现：
- 缩进四个空格（Tab）
- 代码块前后分别用三个反引号（```）包裹
    - 反引号需要单独成行，即与代码不共行
    - 可在第一行三个反引号后紧跟代码语言，可实现语言高亮（这是第一种Tab方法无法做到的）

    return 0;
}

## 引用
用大于号（>）开头，若干大于号对应若干级引用，后面无需加空格。

>这是一级引用
>>二级引用


## 列表
无序列表：用星号（*）、加号（+）或减号（-）开头，需要在后面加上一个空格。如需添加子列表可缩进四个空格（Tab）。

有序列表：用数字加实心圆点（.）开头，需要在后面加上一个空格。数字顺序不影响渲染。

任务列表：用减号（-）+空格+方括号（[]）+空格开头
- 方框内需要加入空格，呈现为空方框
- 如在方括号中写入x则呈现为打勾方框
- 在vscode中如果无法显示任务框，只是显示普通列表通常是插件缺失或预览配置未启用

## 分割线
用多个星号（***）、多个减号（---）或多个下划线（___）表示，至少三个字符。

# 写在最后
即使对于markdown的一些书写方法暂不熟悉，也可以使用vscode创建.md文件自我尝试，点击编辑界面的右上角按钮，可选择**打开侧边预览**实时观察自己的文本编辑效果。

另外，如果想要在github上查看.md文件的原始代码版本，可以打开目标文件后，选择以下方式中的任意一种方式查看：
- 点击右上方的**Raw**按钮（在gitee中对应**原始数据**按钮）
- 点击左上方**Preview**按钮旁的**Code**按钮
